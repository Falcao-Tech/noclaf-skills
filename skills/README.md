# Skills

Skills são **model-invoked**: o agente ativa sozinho quando o contexto bate com a `description` do `SKILL.md`. Ficam em `<seção>/<nome>/SKILL.md`.

## Seções

- **[dev](dev/README.md)** — skills de engenharia (código, review, design), subdivididas por stack.
- **[productivity](productivity/README.md)** — fluxo de trabalho e escrita, fora do código. (vazio)

> A seção é só organização — **não** muda como a skill é invocada; serve pra você e o time acharem a skill, não pro agente. Regra de idioma do repo: skill/command/doc em **pt-BR**, código em **inglês**.
